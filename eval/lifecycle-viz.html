Memesis Memory Lifecycle

How observations are promoted, demoted, and deprecated across stages

Ephemeral
Raw session observations. Created during reduce from transcripts. Short-lived — promoted or pruned during consolidation.
Entry criteria
  • Any observation extracted by reduce
  • No quality gate — everything starts here
Exit triggers
  • Consolidation (LLM decides keep/prune/promote)
  • Deprecation after 30 days of no activity

Consolidated
Validated observations that passed the LLM quality gate. Accumulate reinforcement count as they're seen in more sessions.
Entry criteria
  • LLM consolidation marked "keep" or "promote"
  • Or: direct creation by reduce with count ≥ 1
Promotion criteria (to crystallized)
  • reinforcement_count ≥ 3
  • Spaced reinforcement: reinforcements span ≥ 2 distinct days
  • Purpose: spacing effect prevents flash-in-the-pan patterns
→
3+ reinforcements
across 2+ days
←
10+ injections,
0 usage
Crystallized
Durable patterns confirmed by repetition. Eligible for token-budgeted Tier 2 injection. Query-aware retrieval can surface these.
Entry criteria
  • Promoted from consolidated (3+ spaced reinforcements)
Promotion criteria (to instinctive)
  • importance ≥ 0.85
  • usage_count ≥ 1 (model actually referenced it)
  • Manual curation / self-reflection can also promote
Demotion criteria (back to consolidated)
  • injection_count ≥ 10 AND usage_count = 0
  • "Injected 10 times but never used" = not valuable

Instinctive
Core identity memories. Always injected into every session — no query needed, no token budget. The "this is who you are" layer.
Entry criteria
  • Promoted from crystallized (high importance + proven usage)
  • Or: self-reflection creates directly
Protection
  • Ghost coherence checks validate consistency
  • Self-reflection can demote if no longer accurate
  • Reconsolidation can update content while preserving stage

Feature Flags — Where They Act

habituation_baseline
Acts at: consolidation (pre-LLM)
Suppresses routine events before the LLM sees them. Prevents common-but-uninteresting observations from entering the store. Acts as a noise gate on the input to consolidation.
orienting_detector
Acts at: consolidation (pre-LLM)
Detects novel or surprising events that warrant attention. Biases toward observations that break established patterns. Complements habituation by amplifying the unusual.
somatic_markers
Acts at: consolidation (post-LLM)
Assigns emotional valence and importance based on affect signals. Sets the importance score that later determines promotion eligibility. High-affect events get higher importance.
replay_priority
Acts at: consolidation (pre-LLM)
Sorts observations by salience before the LLM processes them. Affects attention ordering — salient observations appear first in the prompt and get more careful LLM treatment.
prompt_aware_tier2
Acts at: retrieval (Tier 2)
Gates whether the query-aware hybrid RRF path is used when a user prompt is available. Without it, Tier 2 always uses the static importance-sorted path regardless of query.
thompson_sampling
Acts at: retrieval (reranking)
Stochastic reranking using Thompson sampling (Beta distribution). Trades precision for exploration — may push well-ranked memories out of the token budget in favor of less-seen ones.
graph_expansion
Acts at: retrieval (candidate pool)
Adds 1-hop neighbor memories (connected by shared narrative threads or tag co-occurrence) to the candidate pool. Surfaces related context but can displace direct matches under tight budgets.
sm2_spaced_injection
Acts at: retrieval (filtering)
SM-2 scheduling algorithm that puts recently-injected memories on cooldown. Prevents the same memories from appearing every session. Filters candidates before token budget is applied.
saturation_decay
Acts at: lifecycle (relevance scoring)
Decays relevance score of memories that have been injected many times without being used. Part of compute_relevance() in the lifecycle engine, not the retrieval path.
integration_factor
Acts at: lifecycle (relevance scoring)
Boosts relevance for memories that share tags or threads with the current context. Measures how well a memory integrates with the broader memory graph.
provenance_signals
Acts at: retrieval (context formatting)
Adds provenance metadata (when created, how many times reinforced, source sessions) to injected memory context. Affects context size but not which memories are selected.
reconsolidation
Acts at: lifecycle (post-session)
Allows existing memories to be updated with new information from current session, not just reinforced. Enables content evolution without creating new memories.
ghost_coherence
Acts at: lifecycle (validation)
Validates that instinctive-tier self-model claims are coherent with observed behavior. Catches contradictions between what the system "believes" and what it does.
affect_awareness
Acts at: consolidation + retrieval
Tracks emotional valence of sessions and memories. Affects both what gets stored (emotional salience) and what gets retrieved (affect-matching).

All Transition Rules

TransitionDirectionTriggerCriteria
ephemeral → consolidated promote consolidator.py (LLM decision) LLM marks observation as "keep" during PreCompact
ephemeral → archived deprecate lifecycle.get_deprecation_candidates() No injection or usage for 30+ days
consolidated → crystallized promote lifecycle.get_promotion_candidates() reinforcement_count ≥ 3, spanning ≥ 2 distinct days
crystallized → instinctive promote lifecycle.can_promote() + self_reflection importance ≥ 0.85 AND usage_count ≥ 1
crystallized → consolidated demote lifecycle.get_demotion_candidates() injection_count ≥ 10 AND usage_count = 0 (injected but never used)
instinctive → crystallized demote self_reflection / ghost_coherence Self-model contradiction detected, or no longer accurate
any → archived deprecate lifecycle.deprecate() Manual or automated deprecation with rationale
any (content update) reconsolidate reconsolidation.py New session evidence updates content while preserving stage

Key insight from eval: reinforcement count ≠ quality. 35% of top-20 most-reinforced observations are codebase-derivable mechanics, not durable patterns. The promotion criteria (3+ spaced reinforcements) gate on frequency, but need a quality gate too.
