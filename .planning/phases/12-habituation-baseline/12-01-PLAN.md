<objective>
Build a per-project habituation model that tracks event frequency and suppresses routine observations before they reach the consolidation LLM.

Purpose: Reduce token waste by filtering out repetitive, low-signal observations (test runs, file saves, routine commits) that the project sees frequently. This is the inverse of OrientingDetector (Phase 11) -- orienting amplifies HIGH-signal moments, habituation dampens LOW-signal routine.

Output: HabituationModel class with persistence, observation filtering, and integration into the consolidation path.
</objective>

    **HabituationModel class:**
    - `__init__(self, base_dir: Path)` — loads existing habituation.json from base_dir if present, otherwise starts empty
    - `_load(self) -> dict[str, int]` — reads {base_dir}/habituation.json, returns empty dict on missing/corrupt
    - `_save(self)` — writes current counts to {base_dir}/habituation.json atomically
    - `extract_event_signature(self, observation_block: str) -> str` — parses the `## [timestamp] type` header from an observation block. Returns the type string (e.g., "correction", "domain_knowledge"). Returns "untyped" if no type found. Normalizes to lowercase.
    - `get_factor(self, event_type: str) -> float` — returns `1.0 / (1.0 + math.log(count))` where count is the number of times this event_type has been seen. Returns 1.0 for unseen events. Returns 1.0 for all events if feature flag disabled.
    - `record_event(self, event_type: str)` — increments count for event_type and calls _save()
    - `filter_observations(self, ephemeral_content: str, threshold: float = 0.3) -> tuple[str, int]` — splits ephemeral content into observation blocks (split on `## [` pattern), computes habituation_factor for each, removes blocks with factor < threshold. Returns (filtered_content, suppressed_count). Records each event to update counts. When feature flag is disabled, returns content unchanged with 0 suppressed.

    The observation block splitting must handle the markdown format: each observation starts with `## [` and runs until the next `## [` or end of content. The session header line `# Session Observations` should be preserved (not treated as an observation).

    Formula: `habituation_factor = 1.0 / (1.0 + math.log(count))` where count is the stored occurrence count for that event type. At count=0 (novel): factor=1.0. At count=10: factor ~= 0.30. At count=100: factor ~= 0.18.

    Note: Use `math.log` (natural log), not `math.log10`.
