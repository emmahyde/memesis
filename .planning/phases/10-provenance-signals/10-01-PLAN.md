<objective>
Add human-readable provenance metadata to injected Tier 2 memories. Each memory block will show "Established across N sessions over M weeks" (computed from RetrievalLog data) or "First observed [relative time]" for single-session memories, giving the reader a sense of how established the knowledge is.

Purpose: Provenance signals communicate memory confidence without exposing raw scores. A memory observed across 7 sessions over 3 weeks carries more weight than one first observed yesterday.
Output: `_compute_provenance()` method + wiring into `inject_for_session()` Tier 2 loop, guarded by `provenance_signals` flag.

  <action>
    **Write tests first** in a new `TestProvenanceSignals` class in tests/test_retrieval.py. Use the existing `base`, `engine`, `_make_memory` fixtures. Create RetrievalLog entries directly via `RetrievalLog.create(...)` to set up session/timestamp data for provenance computation.

    For the time computation tests, use `monkeypatch` to freeze `datetime.now()` or compute expected values relative to the actual created timestamps.

    **Then implement:**

    1. Add `_compute_provenance_batch(self, memory_ids: list[str]) -> dict[str, str]` to `RetrievalEngine`:
       - Single query: `RetrievalLog.select(RetrievalLog.memory_id, fn.COUNT(fn.DISTINCT(RetrievalLog.session_id)).alias('session_count'), fn.MIN(RetrievalLog.timestamp).alias('earliest')).where(RetrievalLog.memory_id.in_(memory_ids)).group_by(RetrievalLog.memory_id)`
       - For each memory_id in results with session_count > 1: compute weeks = (now - earliest).days // 7, format "Established across {N} sessions over {M} weeks" (use "over less than a week" when weeks == 0)
       - For memory_ids with session_count == 1 or not in results: use Memory.created_at to compute relative time, format "First observed {relative_time}"
       - Relative time logic: < 1 day = "recently", 1 day = "1 day ago", N days = "N days ago", 7-13 days = "1 week ago", etc.
       - Return dict mapping memory_id -> provenance string
       - Import `from peewee import fn` at top of file

    2. Modify the Tier 2 formatting loop in `inject_for_session()` (lines 112-126):
       - Before the for loop, compute provenance batch: `provenance_map = self._compute_provenance_batch([m.id for m in tier2])` if `get_flag("provenance_signals")` is True, else `provenance_map = {}`
       - Inside the for loop, after the title line and before the summary line, insert: `if memory.id in provenance_map: sections.append(f"*{provenance_map[memory.id]}*")`
       - This produces output like:
         ```
         ### Memory Title (importance: 0.85)
         *Established across 7 sessions over 3 weeks*
         *Summary text*
         Content...
         ```

    **Performance:** The batch query approach (one SELECT with GROUP BY) avoids N+1 queries. For 5-15 Tier 2 memories, this is a single indexed query on retrieval_log(memory_id).

    **Edge cases:**
    - Memory with no RetrievalLog entries AND no created_at: return "First observed" with no time qualifier
    - Empty tier2 list: provenance_map = {}, no query issued
    - Provenance flag disabled: skip computation entirely, no query overhead
