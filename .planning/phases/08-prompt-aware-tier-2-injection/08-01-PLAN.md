<objective>
Wire user prompt text into the Tier 2 crystallized retrieval path so that
UserPromptSubmit uses prompt-aware hybrid RRF for crystallized memory
injection, not just the all-stage JIT search.

Purpose: Currently `session_start.py` calls `inject_for_session` with no query
(correct -- no prompt exists yet). When the first user prompt arrives,
`user_prompt_inject.py` runs its own `hybrid_search` across ALL stages but
never calls the Tier 2 API (`get_crystallized_for_context`). This means the
Tier 2 path -- with its project_context boost, token budget, and
crystallized-only filtering -- is never fed prompt text as a retrieval signal.

FOUND-02 requires that the Tier 2 injection path uses prompt text. Phase 7
already built the `query`/`query_embedding` parameters on
`get_crystallized_for_context` and `inject_for_session`. This plan wires the
UserPromptSubmit hook to call that Tier 2 API, closing the last gap.

Output: Modified `user_prompt_inject.py` calling `get_crystallized_for_context`
with prompt-derived query, plus tests proving the wiring.
</objective>
