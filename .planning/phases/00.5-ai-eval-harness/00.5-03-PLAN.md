<task type="auto">
  <name>Task 2: Create eval/verify_phase.py verifier hook</name>
  <files>eval/verify_phase.py</files>
  <action>
Create eval/verify_phase.py as both a CLI script and an importable module.

The verifier:
1. Loads .planning/eval-baseline.json as the reference baseline.
2. Runs eval/capture_baseline.py logic inline (or calls `capture_baseline(phase=phase_name)`) to get current scores.
3. Computes delta = current - baseline for each metric.
4. Prints a formatted table:

```
Phase: 07-after-rrf
Metric                  Baseline   Current    Delta
----------------------------------------------------
retrieval.precision_at_1   0.00      0.67     +0.67  IMPROVED
retrieval.mrr              0.00      0.82     +0.82  IMPROVED
consolidation.prune_f1     0.00      1.00     +1.00  IMPROVED
longmemeval.accuracy       0.00      0.30     +0.30  IMPROVED
```

5. Regression rule: exit code 1 if ANY metric drops more than 0.05 below baseline.
   Print "REGRESSION DETECTED: {metric} dropped {delta:.2f}" for any regression.
   Exit code 0 if no regression (improvement or flat is fine).

CLI usage:
```
python3 eval/verify_phase.py --phase "07-after-rrf"
python3 eval/verify_phase.py  # uses "current" as phase name
```

Importable usage:
```python
from eval.verify_phase import verify_against_baseline
exit_code = verify_against_baseline(phase="07-after-rrf")
```

The script must handle .planning/eval-baseline.json not existing: print a warning and exit 0 (not a regression, just no baseline yet).

Keep the implementation under 120 lines. Import from eval.metrics and eval.longmemeval_adapter. Do NOT import from core.* at module level.
